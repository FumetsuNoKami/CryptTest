# Начало работы

Это руководство поможет вам запустить проект с нуля, даже если вы новичок в Go.

---

## 1. Установите Go

Скачайте и установите Go с официального сайта: https://go.dev/dl/

Проверьте установку:

```bash
go version
# go version go1.22.x ...
```

---

## 2. Что такое go.mod и go.sum?

`go.mod` — это файл-манифест проекта. Он хранит:

- имя модуля (пути к пакетам внутри проекта)
- версию Go
- список зависимостей (внешних библиотек)

`go.sum` — файл с хешами зависимостей для проверки целостности. **Не редактируйте его вручную.**

---

## 3. Загрузите зависимости

```bash
go mod tidy
```

Эта команда:

1. Скачивает все пакеты из `go.mod`
2. Удаляет неиспользуемые записи
3. Обновляет `go.sum`

---

## 4. Создайте файл конфигурации

```bash
cp .env.example .env
```

Откройте `.env` и при желании измените порт или добавьте API-ключ CoinGecko.

> **Бесплатный план CoinGecko** позволяет ~30 запросов в минуту без ключа.  
> Этого достаточно для разработки. Ключ можно не указывать.

---

## 5. Запустите сервер

```bash
go run ./cmd/server/main.go
```

Или через Makefile (если установлен `make`):

```bash
make run
```

Вы увидите в консоли:

```
time=2024-... level=INFO msg="starting crypt server" addr=localhost:8080
time=2024-... level=INFO msg="server listening" addr=localhost:8080
```

---

## 6. Протестируйте API

Откройте новый терминал и выполните:

```bash
# Проверка живости сервера
curl http://localhost:8080/health

# Топ-5 монет
curl "http://localhost:8080/api/v1/coins?per_page=5"

# Bitcoin детально
curl http://localhost:8080/api/v1/coins/bitcoin | python -m json.tool
```

---

## 7. Остановите сервер

Нажмите `Ctrl+C`. Сервер корректно завершит все текущие запросы (graceful shutdown).

---

## 8. Полезные команды Go

| Команда                                      | Описание                  |
| -------------------------------------------- | ------------------------- |
| `go run ./cmd/server/main.go`                | Запустить без компиляции  |
| `go build -o bin/crypt ./cmd/server/main.go` | Скомпилировать в бинарник |
| `go test ./...`                              | Запустить все тесты       |
| `go mod tidy`                                | Обновить зависимости      |
| `go vet ./...`                               | Статический анализ кода   |
| `go fmt ./...`                               | Автоформатирование кода   |

---

## 9. Что такое горутины?

В `main.go` вы видите:

```go
go func() {
    srv.ListenAndServe()
}()
```

`go` перед функцией запускает её в отдельной **горутине** — это лёгкий поток в Go.  
Благодаря этому основной поток продолжает выполняться и может ждать сигнала остановки.
